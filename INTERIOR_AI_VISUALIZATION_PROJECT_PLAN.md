# 홈 스피커 사용환경 시각화 고도화 프로젝트 제안서

## 1. 프로젝트 개요
- **목적**: 비즈컴(생성형 AI) + GPT를 활용해, 홈 스피커 제품 프레젠테이션용 인테리어 사용환경 이미지를 고품질/고속으로 생산하는 체계를 구축한다.
- **핵심 가치**
  - 현실감 있는 결과물 (사실감, 색감, 연출)
  - 브랜드 톤 일관성 유지
  - 카메라/구도/배치 제어성 확보
  - 사용자 개입 최소화(선택형 UI + 자동 조합 운용)
  - 팀 확장 가능한 업무 패키지화

---

## 2. 목표와 성공 기준

### 2.1 산출물 목표
- 작업 원본: 프레임 확장/크롭 내성을 고려한 대형 캔버스(권장 4K 이상)
- 기본 납품본: QHD(2560x1440), 프레젠테이션 비율(16:9)
- 파생 납품본: 발표/문서/썸네일 전환을 위한 범용 비율(1:1, 4:5, 3:2) 크롭본
- 확장 산출물: 제품 강조 컷, 무드 컷, 비교 컷, 필요 시 애니메이션용 시퀀스 프레임
- 공간 범위: 집 내부(거실/침실/서재/주방 인접 공간 등)

### 2.2 품질 목표(우선순위)
1. 현실감
2. 브랜드 톤 적합성
3. 이미지 완성도(재질 표현, 조명, 색감, 연출)

### 2.3 운영 목표
- 기존 방식 대비 생산 속도 향상
- 재작업률 감소
- 반복 가능한 출력 품질 확보
- 타 팀 전파 가능한 교육/운영 문서 확보

---

## 3. 대원칙: “생성”보다 “제어 가능한 생성 시스템”

단발성 프롬프트가 아니라, 다음 4가지 구성요소를 표준화한다.

1. **Scene Spec(장면 명세)**: 공간/구도/무드/금지요소를 구조화
2. **Prompt Pack(프롬프트 팩)**: 목적별 템플릿 + 네거티브 규칙 + 브랜드 상시 가드레일
3. **QC Rubric(검수 기준표)**: 품질 평가의 객관화
4. **Iteration Log(반복 로그)**: 무엇이 좋은 결과를 내는지 축적

## 4. 시스템 아키텍처 (API 없음 가정)

## 4.1 입력 계층 (텍스트 중심)
입력은 자연어가 아닌 **준구조화 텍스트**로 받는다.

예시 필드:
- 제품 타입/스케일: 소형/중형/대형 홈 스피커
- 공간 타입: living_room, bedroom, study
- 페르소나 무드: minimal_warm, modern_luxury, cozy_family
- 지역/연령 맥락: KR_30s, KR_40s 등
- 카메라: eye-level / 35mm / 3-quarter
- 구조 제약: 창문 방향, 벽 방향, 가구 배치
- 제품 위치: center_table / side_table / shelf
- 비워둘 영역(hero zone): 예) 중앙 하단 30%
- 금지 요소: 과도한 텍스트, 비현실적 반사, 왜곡된 가구 비율
- 브랜드 금지 규칙: posed portrait, camera eye contact, 과밀/과장 스타일링, sterile/mono-material 환경

## 4.2 생성 계층 (Bizcom + GPT 보조)
- GPT 역할: 입력값 검증, 프롬프트 자동 조합, 누락값 보정, 브랜드 위반 사전 차단
- 비즈컴 역할: 이미지 생성/변형/세부 보정
- 반복 방식:
  1) 베이스 공간 3안 생성
  2) 제품 배치 가능 구도 선별
  3) 선택안 고정 시드/고정 구도 재생성
  4) 디테일 업그레이드(재질/광원/그림자)

## 4.3 출력 계층
- 마스터본: 4K 이상 고해상도 범용 캔버스(크롭/리프레임 기준 원본)
- 최종본: QHD 16:9
- 파생본: 1:1, 4:5, 3:2 등 목적별 리프레임 산출
- 리뷰본: 저용량 썸네일 + 비교 시트(안 A/B/C)
- 기록물: 장면 명세 + 프롬프트 + 선택 사유 + 보정 이력


## 4.4 브랜드 가드레일 강제 정책 (옵션 아님)
- 브랜드 가이드라인(Embrace/Avoid)은 **항상 기본 적용**한다.
- 사용자 선택 UI에서 브랜드 관련 항목은 토글이 아니라 **락(고정값)** 으로 처리한다.
- 생성 프롬프트는 반드시 아래 3중 구조를 포함한다.
  1) Brand Embrace 문장
  2) Brand Avoid 네거티브 문장
  3) 위반 시 자동 수정 리비전 문장
- QC 단계에서 브랜드 위반 항목 1개라도 발생하면 Fail로 분류한다.

## 5. 제어 포인트 설계

## 5.1 카메라 제어
- 기본 카메라 규칙을 다음으로 고정한다.
  - pitch: **상향 금지**, 하향 15~30도
  - yaw: **정면 0도 지양**, 좌/우 30도 중심
  - distance: 제품+환경 동시 인지가 가능한 미디엄 거리
  - lens/fov: 과왜곡 방지를 위해 35~50mm 상당 시야 우선
- 프리셋은 “각도 중심”으로 단순화한다.
  - C1: down 15 / left 30
  - C2: down 20 / right 30
  - C3: down 30 / left or right 30
- 프리셋별 허용 편차(±5도)만 열어두고, 정면/극단 와이드는 기본 금지한다.

## 5.2 공간 구조 제어
- 창문/벽/가구 방향을 텍스트 규칙으로 명시
  - 예: "main wall = north, window = east, sofa faces south"
- 가구 배치 밀도 규칙
  - 제품 가시성 확보를 위해 전경 clutter 제한

## 5.3 제품 배치 제어
- 스피커 예상 배치 위치를 기본값으로 제공
  - 사이드테이블, TV콘솔, 책장, 아일랜드 주변 등
- “비워둘 영역” 명세 필수
  - 프레젠테이션 합성/확장 대비
- “크롭 안전영역(crop-safe area)” 동시 명세
  - 예: 핵심 제품은 중앙 60% 내 유지, 주변 20%는 잘려도 무관한 배경으로 구성

## 5.4 무드/브랜드 제어
- 브랜드 가이드북 수신 후 다음을 고정값으로 전환
  - 색온도 범위
  - 선호 재질
  - 금지 스타일
  - 라이프스타일 톤(절제/고급/친밀 등)

## 5.5 실무형 캔버스/크롭 전략 (전문가 권장)
- 제작은 “최종 비율”이 아닌 “재활용 가능한 마스터 비율”에서 시작
  - 권장 마스터: 3:2 또는 4:3 기반 고해상도 캔버스
  - 16:9는 최종 전달 단계에서 리프레임
- 장면 설계 시 3중 영역을 의무화
  1) **Hero Zone**: 제품/핵심 연출(절대 보존)
  2) **Safe Zone**: 주요 가구/광원/구도선(가능하면 보존)
  3) **Bleed Zone**: 크롭 가능 여백
- 카메라 프롬프트에 여백 의도 명시
  - 예: "leave generous negative space around subject for multi-aspect crops"
- 프레젠테이션/문서/SNS 재활용을 가정해 한 장면당 다중 비율 미리 검증
  - 최소 검증 세트: 16:9, 1:1, 4:5


## 5.6 입력 UX 전략: 웹 UI가 최선인지 판단 기준
- 결론: 현 단계에서의 최선은 **“웹 UI 단독”이 아니라 “선택형 UI + GPT 변환기” 2단 구조**다.
- 이유
  - 웹 UI만으로는 문장 품질/브랜드 문맥 정제 한계가 있음
  - GPT만으로는 선택 속도/재현성 한계가 있음
  - 결합 시, 입력 속도(UI)와 문장 품질(GPT)을 동시에 확보
- 대안 비교
  1) 순수 채팅 입력: 구축 빠름, 사용성/일관성 낮음
  2) 순수 웹 UI: 선택 속도 높음, 프롬프트 유연성 낮음
  3) UI+GPT(권장): 선택 속도/일관성/품질 균형이 가장 좋음
- 구현 우선순위
  - v1(즉시): 경량 폼 + GPT 변환
  - v2(검증 후): 전용 웹 UI(프리셋 저장/불러오기, 결과 로그 자동화)

## 7. 성과지표(KPI) 설계

정성 + 정량을 함께 본다.

### 7.1 정량 지표
- First Pass 도달시간(Time-to-Pass): 기준 충족 결과 최초 확보 시간
- 제작 리드타임: 최초 시안까지 소요 시간
- 채택률: 생성안 중 채택 비율
- 재작업률: 채택 후 수정 반복 횟수
- Pass까지 재생성 횟수(Regen-to-Pass)
- 컷당 총 작업시간
- 월 생산량(품질 기준 통과컷 기준)
- 멀티 비율 재활용률: 마스터 1컷에서 파생 납품본으로 전환 가능한 비율 수

### 7.2 정성 지표
- 현실감 평가(5점 척도)
- 브랜드 톤 적합도(5점)
- 브랜드 규칙 준수율(위반 컷 비율)
- 프레젠테이션 설득력(5점)
- 크롭 견고성: 비율 전환 후에도 제품 주목도/구도 완성도가 유지되는지 평가

### 7.3 기준선 설정
- 기존 업무방식으로 2~3개 과제 기준선 측정
- 동일 조건으로 AI 파이프라인 비교 후 개선율 계산

## 8. 단계별 실행 로드맵

## 0단계: 준비 (1주)
- 브랜드 가이드 수집
- 홈 스피커 대표 스케일 3종 정의
- 공간/카메라 기본 프리셋 확정

## 1단계: 파일럿 (2주)
- 공간 타입 2종, 무드 2종으로 소규모 실험
- 프롬프트/명세 템플릿 1차 고정
- 브랜드 상시 가드레일(고정 규칙) 잠금 적용
- QC 기준표 적용 및 보정
- 마스터본 1장당 3개 비율(16:9, 1:1, 4:5) 파생 검증

## 2단계: 확장 (2~4주)
- 공간 타입 4~6종 확대
- 제품 배치 위치 다양화
- 애니메이션 시퀀스 실험 시작

## 3단계: 패키지화 (지속)
- 운영 문서/교육 자료 완성
- UI+GPT 2단 입력 체계 운영 가이드 배포
- 신규 사용자 재현성 검증
- 타팀 전파 가능 버전 배포
- “크롭 전략 포함 표준 작업서(SOP)” 배포

## 11. 보고 요약 (의사결정용)
이 프로젝트의 본질은 “이미지 생성 도구 활용”이 아니라, **제품 프레젠테이션용 사용환경 시각화의 운영체계 구축**이다.
- 단기적으로는 시각화 속도/품질을 끌어올리고,
- 중기적으로는 제어성과 일관성을 확보하며,
- 장기적으로는 교육 가능한 조직 자산(업무 패키지)으로 전환한다.

API 없이도 텍스트 중심 표준화를 통해 충분히 고도화 가능하며,
비즈컴 + GPT 조합으로 현재 업무방식 대비 높은 효율 개선을 기대할 수 있다.

## 12. 에이전트 단독 수행 가능 범위와 실행계획

본 섹션은 “내가 혼자 알아서 할 수 있는 것”을 명확히 분리해, 사용자 개입이 필요한 단계와 자동 진행 가능한 단계를 구분한다.

### 12.1 에이전트 단독 수행 가능 항목 (즉시 가능)
1. **요구사항 구조화**
   - 대화 로그를 Scene Spec 필드로 정규화
   - 브랜드 강제 규칙/카메라 규칙/출력 규격을 체크리스트화
2. **문서 체계화**
   - 프로젝트 계획서, SOP, QC Rubric, 로그 템플릿 자동 작성/개정
   - 변경 이력 관리 및 버전 라벨링(v0.1, v0.2)
3. **프롬프트 시스템 설계**
   - 선택형 옵션 사전(레고 블록) 정의
   - GPT 변환용 중간 포맷(JSON) 스키마 설계
   - 생성/네거티브/리비전 프롬프트 템플릿 작성
4. **사용성 검증 설계**
   - Time-to-Pass, Regen-to-Pass 등 KPI 측정 표준 정의
   - 파일럿 실험 시나리오(세션 단위) 설계
5. **실행 리포팅 자동화**
   - 세션 로그 양식 자동 생성
   - 주간 회고 리포트 초안 자동 생성

### 12.2 사용자 확인이 필요한 항목 (의사결정 필요)
1. 브랜드 가이드북의 최종 원문/금지 표현 확정본 반영
2. 비즈컴 실제 생성 결과 선택(최종 Pass 판정)
3. 우선 적용 공간/제품 시나리오의 비즈니스 우선순위
4. UI 구현 방식 확정(노코드 MVP vs 경량 웹앱)

### 12.3 2주 실행계획 (에이전트 주도)

#### Week 1 — 설계 고정
- D1: Scene Spec v1 + Brand Guard v1 + Camera Preset v1 고정
- D2: 옵션 사전(공간/조명/배치/소품/금지요소) 작성
- D3: GPT 변환 프롬프트(생성/네거티브/리비전) 작성
- D4: 세션 로그 템플릿 + KPI 측정표 작성
- D5: 파일럿 운영 매뉴얼(1~2시간 세션용) 완성

#### Week 2 — 사용성 검증
- S1~S2 세션: 동일 장면군으로 반복 테스트
- 측정: Time-to-Pass, Regen-to-Pass, 수동 수정률
- 분석: 실패원인 Top3(각도 미스/배치 미스/톤 미스) 도출
- 개선: 프리셋/네거티브/리비전 템플릿 1회 업데이트

### 12.4 완료 기준(Definition of Done)
- 브랜드 Avoid 위반 없이 Pass 가능한 세션 재현
- 첫 Pass 도달시간 단축 경향 확인
- 재생성 횟수 감소 경향 확인
- 문서 패키지(계획서/SOP/QC/로그) 단일 저장소에서 재사용 가능

## 13. 프롬프트 제조 UI 구현 접근 (우선순위 상세)

사용자 질문의 핵심은 “어떻게 구현할지”이므로, 아래를 **실제 개발 순서** 기준으로 제시한다.

### 13.1 구현 철학
- 목적은 화려한 UI가 아니라 **입력속도 + 제어성 + 재현성** 확보다.
- 따라서 자유 텍스트 입력창은 보조수단으로 두고, 기본은 **레고형 선택 UI**로 고정한다.
- 브랜드 가드레일은 선택 옵션이 아니라 **하드코딩된 정책 계층**으로 분리한다.

### 13.2 정보구조(IA)
UI는 아래 6개 패널만 제공한다.
1. Scene: 공간/시간대/무드
2. Camera: pitch, yaw, distance, lens
3. Product: 스케일/배치면/keep-out zone
4. Props: 소품 밀도/허용군/금지 위치
5. Output: 마스터 비율/파생 비율
6. Result: 조합식(JSON) + GPT 변환 프롬프트 + 복사 버튼

> 핵심: 사용자는 “문장 작성”이 아니라 “옵션 선택”만 수행한다.

### 13.3 데이터 모델 (UI 내부 표준)
모든 선택값은 다음 구조로 저장한다.

```json
{
  "scene": {"room": "living_room", "time": "evening", "mood": "minimal_warm"},
  "camera": {"pitch": 20, "yaw": 30, "distance": "medium", "lens": "35-50"},
  "product": {"scale": "onyx_class", "placement": "side_table", "keep_out": "medium"},
  "props": {"density": "low", "avoid_near_product": true},
  "output": {"master_ratio": "3:2", "derivatives": ["16:9", "1:1", "4:5"]},
  "brand_lock": {"embrace": "required", "avoid": "required"}
}
```

### 13.4 프롬프트 생성 파이프라인
1. **입력 검증**: 각도 범위/충돌값 검사 (예: pitch 0 입력 차단)
2. **정책 주입**: Brand Embrace/Avoid 자동 삽입 (해제 불가)
3. **문장 조합**: Bizcom 생성 프롬프트 + 네거티브 프롬프트 + 리비전 프롬프트 3종 생성
4. **출력 제공**: `Copy Prompt`, `Copy JSON`, `Copy Revision` 버튼 제공

### 13.5 화면/인터랙션 상세
- 좌측: 옵션 패널(아코디언)
- 중앙: 라이브 미리보기(현재 조합 텍스트)
- 우측: 최종 출력(생성/네거티브/리비전 탭)
- 하단: Pass/Fail 체크박스 + 세션 로그 저장 버튼

필수 UX 장치:
- `Preset 불러오기/저장`
- `최근 성공 조합 5개 추천`
- `충돌 경고 배지` (예: clutter high + keep-out large)
- `원클릭 초기화` 및 `원클릭 복사`

### 13.6 기술 구현안 (2단계)
#### v1 (1주, 빠른 검증)
- 경량 웹앱(단일 페이지)
- 로컬 JSON preset 파일 사용
- GPT 변환은 텍스트 복붙 방식으로 시작
- 세션 로그는 Markdown 자동 생성

#### v2 (검증 후)
- preset 버전관리(v1.0/v1.1)
- 실패원인 태깅(각도/배치/톤) 자동 집계
- 성공 조합 자동 추천
- 팀 배포용 읽기 전용 “브랜드 락 모드” 제공

### 13.7 내가 즉시 구현 가능한 범위 (실행 약속)
- UI 필드 스키마 확정
- JSON 조합식 템플릿 제공
- GPT 변환 프롬프트(시스템/유저) 작성
- 세션 로그 자동 템플릿 작성
- Pass/Fail 체크리스트 연결

### 13.8 사용자 확인이 필요한 최소 의사결정
- UI 형태: 노코드 MVP vs 웹앱 MVP
- 우선 preset 10개 목록 확정
- 비즈컴에서 자주 쓰는 리비전 명령 3종 확정

### 13.9 권장 시작점
현재 맥락에서는 **v1 경량 웹앱 + GPT 변환기**가 최적이다.
- 이유: 개발비용이 낮고, 사용성 검증이 빠르며, 이후 확장(v2) 경로가 명확함.
- 목표: 1~2시간 세션에서 Time-to-Pass/ ReGen-to-Pass를 즉시 줄이는 것.

## 14. UI 구현 보고 (실행 기준 상세)

아래는 “지금 바로 만들면 어떻게 만들 것인지”를 개발 태스크 단위로 끊은 구현 보고다.

### 14.1 구현 범위 정의 (MVP)
- 범위 포함
  - 옵션 선택 UI
  - 조합식(JSON) 생성기
  - GPT 변환 입력문 자동 생성기
  - 복사/저장/로그 기록
- 범위 제외(후속)
  - Bizcom API 자동 호출
  - 팀 권한관리/SSO
  - 대시보드형 분석 시각화

### 14.2 기술 스택 제안
- Frontend: Next.js + TypeScript + Tailwind
- State: Zustand(경량 전역 상태)
- Validation: Zod(입력 검증 + 스키마 재사용)
- Storage(v1): LocalStorage + JSON export
- Storage(v2): Supabase(세션 로그/프리셋 버전 저장)

선정 이유:
- 빠른 프로토타입과 유지보수의 균형
- JSON 중심 데이터모델과 타입 안정성 확보
- 후속 확장(v2 API/DB) 경로가 자연스러움

### 14.3 컴포넌트 설계
1. `ScenePanel`
   - room, mood, time 선택
2. `CameraPanel`
   - pitch(15~30), yaw(±30), distance, lens
   - 금지값 입력 시 즉시 경고
3. `ProductPanel`
   - scale, placement, keep-out
4. `PropsPanel`
   - density, 허용/금지 위치
5. `OutputPanel`
   - master ratio(3:2 기본), derivatives
6. `BrandLockPanel` (읽기 전용)
   - Embrace/Avoid 고정 규칙 표시
7. `PromptResultPanel`
   - Bizcom 생성/네거티브/리비전 탭
   - Copy 버튼 3종
8. `SessionLogPanel`
   - Pass/Fail, 메모, 저장

### 14.4 핵심 로직
- `buildSpec(state)`
  - UI 상태를 표준 Scene Spec JSON으로 변환
- `validateSpec(spec)`
  - 각도 범위, 충돌 조합, 필수값 누락 검증
- `injectBrandGuard(spec)`
  - 브랜드 문구 강제 삽입(해제 불가)
- `composePrompt(spec)`
  - 생성/네거티브/리비전 프롬프트 생성
- `saveSession(result)`
  - local log 저장 + markdown export

### 14.5 UX 플로우 (실사용)
1. 프리셋 선택 또는 새 세션 시작
2. 옵션 클릭(약 1~2분)
3. 경고 배지 확인 후 자동 교정
4. 최종 프롬프트 복사 → Bizcom 생성
5. 결과 Pass/Fail 기록
6. 다음 세션에서 성공 프리셋 재사용

### 14.6 품질 게이트 (UI 내부)
- Gate A: 카메라 규칙 위반 차단
- Gate B: 브랜드 락 미포함 시 출력 차단
- Gate C: keep-out과 prop 충돌 시 경고
- Gate D: 파생 비율 미선택 시 확인 모달

### 14.7 1주 구현 일정
- Day 1: 프로젝트 셋업 + 데이터 스키마(Zod)
- Day 2: 패널 UI(1~4) 구현
- Day 3: 출력 패널 + 브랜드 락 패널 구현
- Day 4: 프롬프트 합성 로직 + 복사 기능
- Day 5: 세션 로그/내보내기 + QA 체크

### 14.8 수용 기준 (Acceptance)
- 옵션 선택만으로 30초 내 프롬프트 초안 생성
- 브랜드 Avoid 문구가 항상 포함됨
- 카메라 위반값 입력 불가(또는 경고 후 저장 차단)
- 로그 저장 후 동일 세션 재현 가능

### 14.9 다음 단계 (v2)
- 프리셋 버전관리 + 변경 이력 비교
- 실패 태그 자동 집계(각도/배치/톤)
- 성공 프리셋 추천 점수화
- 팀 배포용 read-only preset pack 제공
